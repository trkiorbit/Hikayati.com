# متجر حكواتي الخارجي — خطة البناء الكاملة
**الدومين:** hikayati.com (Hostinger)
**الهدف:** متجر طباعة حسب الطلب + شحن كريدت، بنفس الهوية البصرية للتطبيق
**آخر تحديث:** 2026-04-18

## 1. الفرق بين المتجر الداخلي والخارجي

| البُعد | داخل التطبيق (IAP) | المتجر الخارجي (hikayati.com) |
|---|---|---|
| **ما يُباع** | باقات الكريدت فقط | منتجات مادية + كروت شحن كريدت |
| **الدفع** | Apple StoreKit + Google Play Billing | Stripe / PayPal / مدى / Apple Pay (ويب) |
| **لماذا منفصل** | Apple/Google يمنعان بيع رقمي بدون IAP | المنتجات المادية **مستثناة** من IAP بسياسة Apple/Google |
| **العمولة** | 15-30% لـ Apple/Google | 2.9% + $0.30 لـ Stripe فقط |
| **التوصيل** | فوري (كريدت في الحساب) | شحن مادي + تأكيد بريد |

**قاعدة ذهبية**: داخل التطبيق لا تذكر المتجر الخارجي بصياغة تربط منتجاً مادياً بمكافأة رقمية (Apple يرفض).

## 6. المعمارية التقنية

### البنية الأساسية (على Hostinger)

```
         - جدول credit_cards

         تستقبل webhook من Stripe → تنشئ order → ترسل إشعار
```

### جداول Supabase الجديدة المطلوبة

```sql
-- طلبات المتجر
CREATE TABLE orders (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id),
  product_type TEXT CHECK (product_type IN ('tshirt', 'booklet', 'magazine', 'credit_card')),
  product_data JSONB,  -- تفاصيل المنتج (مقاس، لون، story_id...)
  quantity INT DEFAULT 1,
  total_sar NUMERIC(10,2),
  status TEXT CHECK (status IN ('pending', 'paid', 'printing', 'shipped', 'delivered', 'cancelled')),
  payment_provider TEXT,  -- stripe / moyasar
  payment_id TEXT,
  shipping_address JSONB,
  tracking_number TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- الفواتير
CREATE TABLE invoices (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  order_id UUID REFERENCES orders(id),
  invoice_number TEXT UNIQUE,
  pdf_url TEXT,
  issued_at TIMESTAMPTZ DEFAULT NOW(),
  total_sar NUMERIC(10,2),
  vat_sar NUMERIC(10,2)
);

-- كروت شحن الكريدت
CREATE TABLE credit_cards (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  code TEXT UNIQUE NOT NULL,
  credits INT NOT NULL,
  price_sar NUMERIC(10,2),
  status TEXT CHECK (status IN ('unused', 'used', 'expired')) DEFAULT 'unused',
  redeemed_by UUID REFERENCES auth.users(id),
  redeemed_at TIMESTAMPTZ,
  order_id UUID REFERENCES orders(id),
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### تدفق الطلب (Order Flow)

```
1. المستخدم في المتجر → يختار منتج
2. يضيف للسلة → Checkout
3. Stripe Checkout Session → بطاقة/Apple Pay
4. Payment success → Stripe webhook
5. Supabase Edge Function:
   a. يُنشئ row في orders (status=paid)
   b. يُنشئ invoice PDF ويرفعه لـ Supabase Storage
   c. يرسل بريد تأكيد للعميل
   d. إذا credit_card → ينشئ code ويُرسله
   e. إذا منتج مادي → يرسل Webhook لشريك الطباعة
6. شريك الطباعة → ينتج ويشحن
7. تحديث status تلقائي عبر API الشريك
```

### Edge Function: stripe-webhook

```typescript
// supabase/functions/stripe-webhook/index.ts
import Stripe from 'https://esm.sh/stripe@14';
import { createClient } from 'https://esm.sh/@supabase/supabase-js@2';

const stripe = new Stripe(Deno.env.get('STRIPE_SECRET_KEY')!);
const supabase = createClient(
  Deno.env.get('SUPABASE_URL')!,
  Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!
);

Deno.serve(async (req) => {
  const sig = req.headers.get('stripe-signature')!;
  const body = await req.text();
  
  const event = stripe.webhooks.constructEvent(
    body, sig, Deno.env.get('STRIPE_WEBHOOK_SECRET')!
  );

  if (event.type === 'checkout.session.completed') {
    const session = event.data.object;
    const metadata = session.metadata;
    
    // 1. Create order
    const { data: order } = await supabase.from('orders').insert({
      user_id: metadata.user_id,
      product_type: metadata.product_type,
      product_data: JSON.parse(metadata.product_data),
      total_sar: session.amount_total / 100,
      status: 'paid',
      payment_provider: 'stripe',
      payment_id: session.payment_intent,
    }).select().single();

    // 2. If credit_card → generate code
    if (metadata.product_type === 'credit_card') {
      const code = generateCode();
      await supabase.from('credit_cards').insert({
        code,
        credits: metadata.credits,
        price_sar: session.amount_total / 100,
        order_id: order.id,
      });
      await sendEmail(session.customer_email, 'credit_card', { code });
    }

    // 3. If physical → notify printer
    if (['tshirt', 'booklet', 'magazine'].includes(metadata.product_type)) {
      await notifyPrinter(order);
    }

    // 4. Generate invoice PDF
    await generateInvoice(order.id);
  }

  return new Response('OK');
});
```

## 10. ملاحظات استراتيجية

### ربح ذكي من كروت الكريدت
- كرت 100 ⭐ بـ 35 ريال = 0.35 ريال/⭐
- نفس الكريدت في App Store بـ $8.99 ≈ 34 ريال لكن Apple يأخذ 30% (24 ريال صافي)
- **المتجر الخارجي يعطيك 35 ريال كاملاً** — ربح أعلى
- ضع كروت بأسعار ترويجية أرخص من App Store → يدفع المستخدمون من المتجر بدل IAP

### عزل Apple/Google بالصياغة
- في التطبيق: **لا تذكر** "اشتر من hikayati.com واحصل على كريدت"
- في المتجر الخارجي: اعرض كروت الكريدت كمنتج مستقل
- **الرابط من التطبيق → المتجر** يجب أن يكون عبر زر بريء: "منتجات حكواتي" بدون وعد بكريدت

### استخدم نفس Supabase
- لا تُنشئ DB منفصلة
- نفس جدول `profiles` يحتوي الرصيد
- استبدال الكرت من داخل التطبيق يزيد `credits` مباشرة
- فواتيرك ومستخدميك في مكان واحد
