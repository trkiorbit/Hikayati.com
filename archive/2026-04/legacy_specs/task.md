# Work Breakdown Structure (WBS) - Hikayati Project

توزيع المهام والمراحل الأساسية لبناء تطبيق حكواتي من الصفر وتجهيزه للإطلاق السريع والآمن.

## Phase 1: Setup & Initialization (تأسيس المشروع والهوية)
- [/] إنشاء مشروع Flutter جديد بالمعمارية اللازمة لـ (MVVM / Clean Architecture).
- [ ] استخراج الشعار والألوان من مجلد `الهوية البصرية` وتطبيقها كملف ثيم (Theme).
- [ ] إعداد خطوط التطبيق (Arabic Typography) ودعم RTL.
- [ ] إعداد مسارات التنقل الأساسية (Routing) بين الشاشات.

## Phase 2: Backend Development (بنية Supabase عبر MCP)
- [ ] إعداد اتصال التطبيق بقاعدة Supabase באמצעות مفاتيح URL والـ Anon Key.
- [ ] إنشاء هيكل جدول `profiles` وجدول `stories`.
- [ ] إنشاء هيكل جدول `public_stories` وجدول `purchases`.
- [ ] تجهيز دوال الإجراءات (RPC) وتحديدًا الدالة الخاصة بالخصم الآمن للرصيد `deduct_credits`.
- [ ] تفعيل إعدادات الخصوصية (RLS - Row Level Security) لجميع الجداول.
- [ ] بناء وحدات التخزين (Storage Buckets) لرفع وحفظ الصور التوليدية والصوت.

## Phase 3: AI Services Engine (محرك الذكاء الاصطناعي للقصص)
- [ ] كتابة خدمة توليد نصوص القصص وهندسة المشاهد (عبر Pollinations Text).
- [ ] كتابة خدمة توليد الصور مع بروتوكول ثبات الأفاتار (عبر Pollinations Image).
- [ ] كتابة خدمة التوليد الصوتي القياسي للذكور والإناث (عبر Pollinations Audio).
- [ ] دمج خدمة استنساخ الصوت مع فحوصات صلاحية ولي الأمر (عبر ElevenLabs).
- [ ] تصميم المساعد الداخلي "حكيم" للاقتراحات والمساعدة (عبر Gemini).

## Phase 4: App Screens & Logic (الشاشات وحركة التطبيق)
- [ ] تطوير واجهة الصفحة الرئيسية (اللافتات، زر الإنشاء، المكتبة العامة).
- [ ] تطوير شاشة "إنشاء قصة" بكامل متغيراتها (اسم الطفل، العمر، الراوي، نوع الموسيقى والقصة، الأفاتار).
- [ ] تطوير "شاشة السينما" التفاعلية (دمج الصوت والصورة، شريط التقدم، تأثيرات فتح واختتام القصة).
- [ ] تطوير صفحة "المكتبة الخاصة" وعداد القفل الذهبي.
- [ ] تطوير صفحة الحساب الشخصي (إحصائيات الكريدت والمقاطع الصوتية المستنسخة).

## Phase 5: Testing & Production (الفحص النهائي والإطلاق)
- [ ] فحص تكامل الذكاء الاصطناعي والتأكد من مطابقة الردود للمواصفات المطلوبة.
- [ ] فحص نظام הכريدت وتجربة الشراء الافتراضية.
- [ ] التأكد من توافق التطبيق مع سياسات المتجر وخصوصية الأطفال.
- [ ] تجهيز التطبيق لنسخ الإنتاج (Release builds).
