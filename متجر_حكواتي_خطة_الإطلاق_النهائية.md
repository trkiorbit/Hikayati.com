## ❷ Supabase — نفس المشروع بدون نقاش

**لا تنشئ مشروع Supabase جديد.**

الجداول الموجودة مسبقاً تُستخدم مباشرة:
- `profiles` ← يحتوي credits، user_id
- `stories` ← لطلبات الكتيبات (story_id)
- `purchases` ← للمشتريات الرقمية

**جداول تُضاف للمتجر الخارجي:**

```sql
-- جدول الطلبات
CREATE TABLE public.orders (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id),
  guest_email TEXT,
  product_type TEXT CHECK (product_type IN ('tshirt','booklet','magazine','credit_card')),
  product_data JSONB,
  quantity INT DEFAULT 1,
  subtotal_sar NUMERIC(10,2),
  vat_sar NUMERIC(10,2) GENERATED ALWAYS AS (subtotal_sar * 0.15) STORED,
  total_sar NUMERIC(10,2) GENERATED ALWAYS AS (subtotal_sar * 1.15) STORED,
  status TEXT CHECK (status IN ('pending','paid','printing','shipped','delivered','cancelled')) DEFAULT 'pending',
  payment_provider TEXT CHECK (payment_provider IN ('moyasar','stripe')),
  payment_id TEXT,
  shipping_address JSONB,
  tracking_number TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
ALTER TABLE public.orders ENABLE ROW LEVEL SECURITY;
CREATE POLICY "User sees own orders" ON public.orders FOR SELECT USING (auth.uid() = user_id);
CREATE POLICY "Guest order insert" ON public.orders FOR INSERT WITH CHECK (true);

-- جدول الفواتير
CREATE TABLE public.invoices (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  order_id UUID REFERENCES public.orders(id) ON DELETE CASCADE,
  invoice_number TEXT UNIQUE,
  pdf_url TEXT,
  total_sar NUMERIC(10,2),
  vat_sar NUMERIC(10,2),
  issued_at TIMESTAMPTZ DEFAULT NOW()
);
ALTER TABLE public.invoices ENABLE ROW LEVEL SECURITY;
CREATE POLICY "User sees own invoices" ON public.invoices FOR SELECT
  USING (order_id IN (SELECT id FROM public.orders WHERE user_id = auth.uid()));

-- جدول كروت شحن الكريدت
CREATE TABLE public.credit_cards (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  code TEXT UNIQUE NOT NULL,
  credits INT NOT NULL,
  price_sar NUMERIC(10,2),
  status TEXT CHECK (status IN ('unused','used','expired')) DEFAULT 'unused',
  redeemed_by UUID REFERENCES auth.users(id),
  redeemed_at TIMESTAMPTZ,
  order_id UUID REFERENCES public.orders(id),
  expires_at TIMESTAMPTZ DEFAULT (NOW() + INTERVAL '1 year'),
  created_at TIMESTAMPTZ DEFAULT NOW()
);
ALTER TABLE public.credit_cards ENABLE ROW LEVEL SECURITY;
CREATE POLICY "User redeems own card" ON public.credit_cards FOR UPDATE
  USING (status = 'unused') WITH CHECK (auth.uid() = redeemed_by);
CREATE POLICY "Service role full access" ON public.credit_cards USING (true);
```

## ❻ تدفق الطلب الكامل (Order Flow)

```
المستخدم يختار منتج
        ↓
يضيفه للسلة (localStorage)
        ↓
Checkout → يدخل بيانات الشحن
        ↓
يختار وسيلة الدفع
   ├── Moyasar → Hosted Checkout → Redirect back
   └── Stripe → Stripe Checkout → Redirect back
        ↓
Payment Success → Webhook يُطلق
        ↓
Edge Function تعمل:
   ① يُنشئ row في orders (status=paid)
   ② يُنشئ فاتورة PDF → Storage
   ③ بريد تأكيد الطلب → العميل
   ④ إذا credit_card:
      - يُنشئ كود 12 خانة
      - بريد مع الكود
   ⑤ إذا منتج مادي:
      - Webhook → شريك الطباعة
        ↓
شريك الطباعة ينتج ويشحن
        ↓
API الشريك يحدث status → shipped
        ↓
بريد "تم شحن طلبك" + رقم التتبع
```

## ❼ Edge Functions

### 1. moyasar-webhook

```typescript
// supabase/functions/moyasar-webhook/index.ts
import { createClient } from 'https://esm.sh/@supabase/supabase-js@2';

const supabase = createClient(
  Deno.env.get('SUPABASE_URL')!,
  Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!
);

Deno.serve(async (req) => {
  const body = await req.json();

  if (body.type === 'payment' && body.status === 'paid') {
    const metadata = body.metadata;

    const { data: order } = await supabase
      .from('orders')
      .insert({
        user_id: metadata.user_id || null,
        guest_email: metadata.guest_email || null,
        product_type: metadata.product_type,
        product_data: JSON.parse(metadata.product_data),
        subtotal_sar: body.amount / 100,
        quantity: metadata.quantity || 1,
        status: 'paid',
        payment_provider: 'moyasar',
        payment_id: body.id,
        shipping_address: JSON.parse(metadata.shipping_address || '{}'),
      })
      .select()
      .single();

    if (metadata.product_type === 'credit_card') {
      const code = generateCode();
      await supabase.from('credit_cards').insert({
        code,
        credits: parseInt(metadata.credits),
        price_sar: body.amount / 100,
        order_id: order.id,
      });
      await sendEmail(metadata.guest_email || metadata.user_email, 'credit_code', { code });
    } else {
      await notifyPrinter(order);
    }

    await generateInvoicePDF(order.id);
  }

  return new Response('OK');
});

function generateCode(): string {
  const chars = 'ABCDEFGHJKLMNPQRSTUVWXYZ23456789';
  const segments = [];
  for (let i = 0; i < 3; i++) {
    let seg = '';
    for (let j = 0; j < 4; j++) {
      seg += chars[Math.floor(Math.random() * chars.length)];
    }
    segments.push(seg);
  }
  return segments.join('-');
}
```

### 2. redeem-credit-card

```typescript
// supabase/functions/redeem-credit-card/index.ts
import { createClient } from 'https://esm.sh/@supabase/supabase-js@2';

Deno.serve(async (req) => {
  const { code, user_id } = await req.json();

  const supabase = createClient(
    Deno.env.get('SUPABASE_URL')!,
    Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!
  );

  // التحقق من الكود
  const { data: card, error } = await supabase
    .from('credit_cards')
    .select('*')
    .eq('code', code.toUpperCase())
    .eq('status', 'unused')
    .single();

  if (error || !card) {
    return new Response(JSON.stringify({ error: 'كود غير صالح أو مستخدم مسبقاً' }), { status: 400 });
  }

  if (card.expires_at && new Date(card.expires_at) < new Date()) {
    return new Response(JSON.stringify({ error: 'انتهت صلاحية الكرت' }), { status: 400 });
  }

  // تحديث الكرت
  await supabase.from('credit_cards').update({
    status: 'used',
    redeemed_by: user_id,
    redeemed_at: new Date().toISOString(),
  }).eq('id', card.id);

  // إضافة الكريدت للمستخدم
  await supabase.rpc('deduct_credits', {
    p_user_id: user_id,
    p_amount: -card.credits,  // سالب = إضافة
    p_reason: `credit_card_redeem:${code}`,
  });

  return new Response(JSON.stringify({
    success: true,
    credits_added: card.credits,
    message: `تم إضافة ${card.credits} ⭐ لحسابك`,
  }));
});
```

## ⓫ ملاحظات استراتيجية نهائية

**كروت الكريدت = أهم منتج لأسباب:**
- يُخفف الاعتماد على App Store (30% عمولة)
- يخدم من ليس لديهم بطاقة ائتمانية
- يمكن بيعه في متاجر جملة مستقبلاً

**شريك الطباعة — اختر واحداً قبل الإطلاق:**
- **Printful** (موثوق، واجهة API سهلة، لكن شحن دولي أطول)
- **Printify** (أرخص، شبكة موردين أكبر)
- **محلي سعودي** (شحن أسرع، لكن يحتاج تكامل يدوي)

**نصيحة:** أطلق الكتيبات والكروت فقط في الأول (بدون شريك طباعة خارجي)، ثم أضف التيشيرتات لاحقاً بعد اختيار الشريك.

**ترتيب الإطلاق المقترح:**
1. كروت الكريدت (لا شريك طباعة، لا شحن — أسهل منتج)
2. الكتيبات (شريك طباعة واحد فقط)
3. التيشيرتات
4. المجلة الشهرية
