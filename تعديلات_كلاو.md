## 2026-04-25 (دفعة 4) — إصلاح صوت assets + زر إغلاق السينما

### السياق
بعد نشر صوت ليلى عبر `publish_pack.ps1`، الصوت لم يخرج عند الفتح من المكتبة العامة. وزر إغلاق السينما كان يحوّل المستخدم إلى المكتبة الخاصة دائماً (حتى لو جاء من العامة).

### التشخيص
- في `cinema_screen.dart` دالة `_playSavedAudio` كانت تستخدم `UrlSource(url)` فقط. مساراتنا الجديدة `assets/public_library/...` تحتاج `AssetSource` لأنها أصول محلية، ليست HTTP.
- زر الإغلاق (السطور القديمة 268-279) كان: `if (fromLibrary) context.go('/private-library')` بغض النظر عن مصدر الفتح. التعليق "✅ المسار الصحيح" مضلل.

### الإصلاحان (محدودان جداً، ملف واحد فقط)
1. **`_playSavedAudio`:** تمييز ثلاثي:
   - `assets/...` → `AssetSource(url.substring('assets/'.length))` (Flutter يضيف assets/ تلقائياً عند تحميل AssetSource).
   - `http://` أو `https://` → `UrlSource(url)`.
   - غير ذلك → log + return بدون crash.
   - logs واضحة: `[Cinema] 🎵 playing asset audio:` / `[Cinema] 🎵 playing network audio:`.
2. **زر إغلاق السينما + زر "العودة" عند المشاهد الفارغة:**
   - `if (context.canPop()) context.pop() else context.go('/')`.
   - `context.pop()` يرجع لنفس الشاشة في الـ navigation stack — أي المكتبة العامة أو الخاصة بحسب المصدر فعلياً (لأن كلتيهما تستخدم `context.push('/cinema', ...)`).
   - الـ fallback آمن للحالات غير المتوقعة (deep link، إعادة فتح، إلخ).
