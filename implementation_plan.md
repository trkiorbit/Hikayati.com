# خطة التنفيذ المتكاملة لمشروع أدب الأطفال (Hikayati)

تهدف هذه الخطة إلى توضيح مسار العمل لبناء وتطوير تطبيق "حكواتي"، موثقة بالاعتماد على الملفات المرجعية المرفقة لاستيعاب الرؤية وضمان أعلى مستويات الدقة والسرعة. وتم التخطيط لتوزيع العمل بين مساعدين اختصاصيين لتبسيط الإدارة وتسريع الإنجاز.

## User Review Required
> [!IMPORTANT]
> يرجى مراجعة هيكل تقسيم العمل (WBS) وتوزيع المهام على المساعدين، وإخباري إذا كنت موافقاً على هذه الخطة لننتقل لاحقاً إلى مرحلة التنفيذ (EXECUTION) للبدء إما بتهيئة البنية التحتية (Supabase) أو واجهات التطبيق (Flutter).

## Proposed Changes

### 1️⃣ تهيئة المشروع وبناء الهوية البصرية (مساعد متخصص Front-End)
- بناء أساس تطبيق Flutter جديد يدعم المعمارية الحديثة.
- دمج الهوية البصرية من المجلد المرفق `الهوية البصرية` (الشعار، الألوان: البنفسجي، الذهبي، و الأبيض النجمي).
- إعداد الخطوط العربية وتجهيز تصميم RTL (من اليمين لليسار).

### 2️⃣ معمارية قواعد البيانات والخلفية - Supabase (مساعد متخصص Backend & MCP)
- استخدام أداة MCP للاتصال ببيئة Supabase المحددة في المشروع وتطويرها.
- **الجداول (Schema):** إنشاء الجداول الأربعة الرئيسية `profiles`، `stories`، `public_stories`، `purchases`.
- **أمن البيانات (RLS):** تطبيق قواعد أمان صارمة لحماية حريات وبيانات الأطفال.
- **الدوال (RPC):** استدعاء وبرمجة دالة `deduct_credits` لإدارة الرصيد بأمان كامل.
- **التخزين (Storage):** تجهيز بيئات حفظ للصور والملفات الصوتية وتحديد صلاحيات الوصول.

### 3️⃣ دمج الذكاء الاصطناعي (مساعد متخصص API & AI)
- دمج واجهة **Pollinations** (لإنتاج النصوص، والمشاهد المصورة بأنماطها المختلفة، والتعليق الصوتي الأساسي).
- دمج واجهة **ElevenLabs** (لاستنساخ أصوات الآباء وإعداد قيود الأمان لعدم استخدامها للأطفال القصر).
- دمج **Gemini API** (لمساعد "حكيم" لتقديم الدعم واقتراح القصص).
- بناء بروتوكول هندسة الأوامر (Prompt Engineering) لضمان ثبات وتجانس تفاصيل ومظاهر الشخصية (Avatar) في كل صور مشاهد القصة.

### 4️⃣ بناء تجربة المستخدم والشاشات (مساعد متخصص UI/UX)
- **الشاشة الرئيسية والمكتبة العامة:** لعرض القصص المعدة مسبقاً وتسهيل الشراء.
- **شاشة إنشاء القصة:** النموذج التفاعلي العميق (أخذ بيانات الطفل، العمر، نوع القصة، الراوي، وغيرها).
- **التجربة السينمائية (شاشة السينما):** عرض القصة بطريقة دمج الصور، النصوص، الحركة الخفيفة، والصوت المرافق كأنها فيلم قصير.
- **المكتبة الخاصة والحساب:** أرشيف محمي بحفظ الإنجازات وشراء السلع المعنوية والمادية وتتبع الكريدت.

## Verification Plan
### Automated Tests
- فحص دوال `RPC` في Supabase عبر استدعاءات برمجية لضمان أن دالة `deduct_credits` لا تخصم بالسالب وأن الرصيد يحسب بدقة.
- اختبار تجاوب اتصال APIs الخاصة بالذكاء الاصطناعي (Pollinations, Elevenlabs) للتحقق من إرجاع مخطط الاستجابة الصحيح (JSON).

### Manual Verification
- إطلاق وتشغيل بيئة التطبيق (Flutter) على المحاكي للتأكد من انسيابية الواجهات وتوافق الهوية البصرية مع شعار "حكواتي".
- تجربة سيناريو الشراء مرورًا بإنشاء القصة لمعاينة الاستهلاك الفعلي للكريدت، وعرض النتيجة النهائية في "شاشة السينما".
