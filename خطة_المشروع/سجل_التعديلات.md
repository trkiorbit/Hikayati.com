## 📌 تعديل 2026-04-24 — batch-002-phase1-auth-decisions ✅

**النوع:** Auth decisions lock — الحد الأدنى الآمن

### الملفات المعدّلة
- `lib/application/use_cases/auth_use_cases.dart`
- `lib/core/auth/auth_service.dart`

### ما تغيّر

#### 1. Eager Profile Creation
**قبل:** profile يُنشأ lazy في `unified_engine.dart` عند حفظ أول قصة عبر `ensureProfileExists`
**بعد:** `AuthUseCases.signUpEmail()` يستدعي `SupabaseService.ensureProfileExists(userId)` مباشرة بعد signup ناجح

```dart
// في signUpEmail بعد التعديل:
final response = await _authService.signUpEmail(email, password);
final userId = response.user?.id;
if (userId != null) {
  await SupabaseService.ensureProfileExists(userId);
}
```

**لماذا:**
- ضمان وجود profile row لكل مستخدم منذ التسجيل
- تجنّب race condition عند أول قصة
- `ensureProfileExists` يتعامل مع duplicate (لا crash إذا موجود)

#### 2. Google Sign-In Disabled رسمياً
**قبل:** Method `signInWithGoogle()` يرمي exception عام، والـ logic خلفه معرّض لـ null crashes
**بعد:** Method يرمي exception واضح `[DISABLED_FOR_FIRST_LAUNCH]` في السطر الأول، والـ logic القديم محفوظ كـ comments للمرحلة 6

**التحقق من UI:**
- `LoginScreen` **لا يحتوي** زر Google أصلاً (فحص مؤكّد)
- لا يوجد استدعاء لـ `signInWithGoogle()` من أي مكان في الـ UI

**تعامل مع الـ import:**
- `google_sign_in` import أصبح unused لكن مُحتفظ به بـ `// ignore: unused_import` مع تعليق "محفوظ لـ Phase 6"
