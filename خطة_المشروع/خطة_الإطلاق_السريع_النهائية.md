### Phase 4 — Orders + Invoices + Redeem Automation

**الهدف:** الأتمتة الكاملة من البيع → الطلب → الفاتورة → البريد → استبدال الكرت.

**يُبنى:**

**Supabase Tables (SQL):**
```sql
CREATE TABLE orders (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES auth.users(id) NULL,  -- NULL إذا زائر
  email TEXT NOT NULL,
  product_type TEXT,  -- tshirt/booklet/magazine/credit_card
  product_data JSONB,
  quantity INT DEFAULT 1,
  total_sar NUMERIC(10,2),
  status TEXT DEFAULT 'pending',  -- pending/paid/processing/shipped/delivered/cancelled
  payment_provider TEXT,
  payment_id TEXT,
  shipping_address JSONB,
  tracking_number TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE invoices (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  order_id UUID REFERENCES orders(id) UNIQUE,
  invoice_number TEXT UNIQUE,  -- HKY-2026-000001
  pdf_url TEXT,
  total_sar NUMERIC(10,2),
  vat_sar NUMERIC(10,2),
  issued_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE credit_cards (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  code TEXT UNIQUE NOT NULL,
  credits INT NOT NULL,
  price_sar NUMERIC(10,2),
  status TEXT DEFAULT 'unused',  -- unused/used/expired
  order_id UUID REFERENCES orders(id),
  redeemed_by UUID REFERENCES auth.users(id),
  redeemed_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_orders_user ON orders(user_id);
CREATE INDEX idx_orders_status ON orders(status);
CREATE INDEX idx_credit_cards_code ON credit_cards(code);
CREATE INDEX idx_credit_cards_status ON credit_cards(status);

-- RLS Policies
ALTER TABLE orders ENABLE ROW LEVEL SECURITY;
ALTER TABLE invoices ENABLE ROW LEVEL SECURITY;
ALTER TABLE credit_cards ENABLE ROW LEVEL SECURITY;

CREATE POLICY "users read own orders" ON orders FOR SELECT USING (auth.uid() = user_id);
CREATE POLICY "users read own invoices" ON invoices FOR SELECT USING (
  order_id IN (SELECT id FROM orders WHERE user_id = auth.uid())
);
```

**Supabase Edge Functions:**
- `payment-webhook` — يستقبل webhook من البوابة، يُنشئ order
- `generate-invoice` — ينتج PDF ويرفعه Storage
- `redeem-card` — يتحقق من الكود ويضيف الكريدت
- `send-email` — يرسل البريد عبر Resend/Brevo
- `notify-printer` — يرسل للشريك عند product_type ∈ {tshirt, booklet, magazine}

**Email Templates (Resend/Brevo):**
- `order-confirmation.html`
- `credit-card-delivered.html` (مع الكود)
- `shipping-notification.html`
- `invoice-email.html` (مع PDF attachment)

**ملفات متأثرة:**
- `supabase/functions/` — 5 Edge Functions
- `supabase/migrations/` — SQL migrations
- Flutter: تحديث orders_screen + redeem_card_screen للاتصال بالـ functions

**ممنوع اللمس:**

- كود توليد القصة
