## كيفية إنشاء نقطة استعادة

عند انتهاء كل Phase بنجاح:

```bash
# 1. التأكد من أن كل شيء مُلتزم
git status   # يجب أن يكون nothing to commit

# 2. إنشاء tag موقّع بوصف واضح
git tag -a rp-XX-name -m "Phase X complete: <description>"

# 3. دفع الـ tag للـ remote
git push origin rp-XX-name

# 4. توثيق النقطة في هذا الملف
```

للعودة لنقطة استعادة:
```bash
git checkout rp-XX-name   # يدخل detached HEAD
# ثم للعودة للـ main:
git checkout main
```

## قواعد استخدام نقاط الاستعادة

### متى تُنشأ نقطة استعادة
- ✅ عند اكتمال phase كاملة بنجاح
- ✅ عند نجاح كل Acceptance Criteria
- ✅ عند نجاح `flutter build apk --debug`
- ✅ عند صفر errors في `flutter analyze`

### متى لا تُنشأ نقطة استعادة
- ❌ عند اكتمال batch واحد فقط (استخدم commit regular)
- ❌ عند كود يعمل جزئياً
- ❌ قبل الاختبار اليدوي

### نظام تسمية Commits بين النقاط
```
feat(phase-N): <batch-name> — <brief>
fix(phase-N): <what-was-fixed>
chore(phase-N): <cleanup>
docs(phase-N): <docs-update>
```

مثال:
```
feat(phase-2): add redeem card screen UI
fix(phase-2): correct Product ID format
docs(phase-2): update test checklist for redeem flow
```

## حماية نقاط الاستعادة

- جميع tags تُدفع لـ origin (GitHub)
- لا تُحذف tags بدون موافقة صريحة
- في حال الحذف الخاطئ: `git tag rp-XX-name <commit-hash>` لاستعادتها محلياً
- Git garbage collection لا يمسح commits مربوطة بـ tags
