## المنتج 1: كروت شحن الكريدت ⭐

**في سلة:**
```
اسم المنتج: كرت شحن حكواتي — 100 ⭐
النوع: منتج رقمي
الوزن: 0
الشحن: غير مطلوب
السعر: 35 ريال
```

| الباقة | الكريدت | السعر |
|---|---|---|
| Mini | +50 ⭐ | 19 ريال |
| Starter | +100 ⭐ | 35 ريال |
| Pro | +250 ⭐ | 79 ريال |
| Ultimate | +500 ⭐ | 149 ريال |

**بعد الشراء (Webhook → Supabase Edge Function):**
```typescript
// سلة تُرسل webhook عند نجاح الطلب
// Edge Function في Supabase تستقبله وتعمل:

1. INSERT في orders (status=paid, product_type=credit_card)
2. توليد كود 12 خانة: XXXX-XXXX-XXXX
3. INSERT في credit_cards (code, credits, order_id)
4. إرسال بريد للعميل بالكود (Resend)
5. إرسال إشعار Push للتطبيق (اختياري)
```

**استبدال الكرت في التطبيق:**
```
التطبيق → المتجر → "لديك كرت شحن؟"
→ يدخل الكود
→ Edge Function: تحقق + تحديث credits في profiles
→ رسالة نجاح: "تم إضافة 100 ⭐ لرصيدك\!"
```

## المنتج 2: كتيب القصة المطبوع 📖

**في سلة:**
```
اسم المنتج: كتيب قصة طفلك — مطبوع
النوع: منتج مادي
السعر: 59 ريال
الشحن: 15 ريال (أو مجاني مع طلب >100 ريال)
وقت التسليم: 5-7 أيام
```

**المتغيرات في سلة:**
```
المتغير 1: اسم البطل (نص حر)
المتغير 2: رابط القصة من التطبيق (أو يُرفع ملف PDF)
```

**بعد الشراء (Webhook → Print.sa API):**
```typescript
// Edge Function تستقبل webhook من سلة:

1. INSERT في orders (status=paid, product_type=booklet)
2. يجلب ملفات القصة من Supabase Storage
3. يرسل للـ Print.sa API:
   {
     "product": "booklet_a5",
     "files": ["cover.pdf", "pages.pdf"],
     "shipping": { address }
   }
4. Print.sa يرد بـ order_id
5. يحفظ print_order_id في orders
6. يرسل بريد تأكيد للعميل
```
